# Triage Labels

The skills speak in terms of triage roles: two **category** roles and five **state** roles. Every triaged issue should carry exactly one of each. This file maps those roles to the actual label strings used in this repo's issue tracker.

## Category roles

| Label in mattpocock/skills | Label in our tracker | Meaning                    |
| -------------------------- | -------------------- | -------------------------- |
| `bug`                      | `bug`                | Something is broken        |
| `enhancement`              | `enhancement`        | New feature or improvement |

## State roles

| Label in mattpocock/skills | Label in our tracker | Meaning                                  |
| -------------------------- | -------------------- | ---------------------------------------- |
| `needs-triage`             | `needs-triage`       | Maintainer needs to evaluate this issue  |
| `needs-info`               | `needs-info`         | Waiting on reporter for more information |
| `ready-for-agent`          | `ready-for-agent`    | Fully specified, ready for an AFK agent  |
| `ready-for-human`          | `ready-for-human`    | Requires human implementation            |
| `wontfix`                  | `wontfix`            | Will not be actioned                     |

When a skill mentions a role (e.g. "apply the AFK-ready triage label"), use the corresponding label string from these tables.

Edit the right-hand columns to match whatever vocabulary you actually use.
